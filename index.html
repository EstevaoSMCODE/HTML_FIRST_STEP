O que são Clients?

Voltar

No linguajar WEB os clients são os "consumidores". Somos nós usuários que estamos navengando por uma rede em busca de uma informação geradas pelo servidor. Quando um computador acessa um site por exemplo, o proprio servidor recebe essa solicitação, processa, e envia para o computador os arquivos estáticos(HTML,CSS,JAVASCRIPT,IMAGENS...) e o próprio navegador gera o chamado cache, onde grava esses arquivos no computador do Client.

Pensando pelo lado do Client, é necessário pensar na Responsibilidade e Compatilidade. Adaptar meu projeto para ser acessado de diferentes dispositivos e navegadores(Browsers) por meio de boas práticas e testes.

O que são Servers

Voltar

Servers são sistemas que provem as informação para o Client. São mais robustos, devido a alta demanda de solicitações e processos que recebem constantemente. Quando um server recebe uma requisição HTTP(cliente), o sistema operacional envia para o servidor web e, em seguida retorna com os arquivos estáticos para o client.

Um conjunto de servidores são chamados de DataCenters

Aqui são alguns dos sistemas operacionais para servidores:

  • Ubuntu
  • Windows Server
  • Fedora
  • Oracle

Aqui são alguns dos servidores web:

  • Apache
  • Nginx

HTML é uma linguagem de programação ?

Voltar

HTML (HyperText Markup Language) não é uma linguagem de programação. Ela é definida como um arquivo de marcação; onde cada delimitador tem sua especificação.

Como é uma estrutura básica de um HTML

Sua estrutura é definida em:

  • < HEAD>

    • Tag onde tudo que será carregado apareçará antes de ser apresentado para o usuário.

  • < BODY>

    • Tag onde tudo aparecerá para o usuário.

  • TAG

    • São "escopos" onde cada um fará uma determinada ação sobre os dados definidos.

  • Atributos das TAG

    • Propriedades que cada TAG pode ter. Podendo ser genéricas ou especificas.

Entendendo a diferença entre listas ordenadas e não ordenadas

Voltar
  • LISTA ORDENADA

    • Para iniciar usamos o < OL> para definir, e dentro dessa tag usamos < li> para registrar cada parametro dessa lista.

  • LISTA NÃO ORDENADA

    • Para inciar usamos o < UL> para definir, e dentro dessa tag usamos novamente o < li> para registrar cada parametro da lista.

A principal diferença dentre uma lista ordenada e não ordenada está na sua criação onde o "OL" é para a ordenada e "UL" para criação da não ordenada
